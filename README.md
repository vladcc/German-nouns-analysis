## German-nouns-analysis
Suffix to gender correlation

### Purpose
To create the shortest list of suffixes with the higher chance of correctness.

### Data
`de-nouns-full.txt` has 371859 nouns. Out of those 47049 suffixes between 1 and
5 letters long are extracted in `sfx-list-full.csv`.

Similarly, 2507 suffixes in `sfx-list-most-common.csv` are extracted from the
1998 nouns in `de-nouns-most-common.txt`.

### How it works
The report takes in 5 parameters, which all have default values.

```
Parameter 1: SuffixList      = sfx-list-full.csv
Parameter 2: SuffixPercent   >= 0.1
Parameter 3: GenderPercent   >= 66
Parameter 4: ShortestSfxOnly =  1
Parameter 5: SortBySuffix    =  0
```

The values can be changed from the command line. If the empty string is given as
a value, the default is taken.

`SuffixPercent   >= 0.1` means a suffix must account for at least 0.1% of all
suffixes.

`GenderPercent   >= 66` means that at least 66% of this suffix must count
towards one gender.

```

38  -ion    3    8804   0.48    51    8515   238   0.58   96.72  2.70
39  -tion   4    7100   0.38    1     7083   16    0.01   99.76  0.23

41  -ktion  5    1791   0.10    0     1790   1     0.00   99.94  0.06
```

one can start by remembering only:
```
37  -on     2    11111  0.60    903   8688   1520  8.13   78.19  13.68
```
because it's still ~80% feminine.


```
38  -ion    3    8804   0.48    51    8515   238   0.58   96.72  2.70
```
is also appealing because the probability jumps to 96% for the cost of 173
nouns less.

Longer suffixes like:
```
40  -ation  5    4227   0.23    1     4213   13    0.02   99.67  0.31
41  -ktion  5    1791   0.10    0     1790   1     0.00   99.94  0.06
```
are probably not worth the bother because even though they are more accurate,
it's more to remember and they are subsets of the shorter ones anyway.
